# GAN Trading — cloud deployment (Render)

Team-accessible version of the trading tool (PZU positions, PI live, performance), with named
logins and mobile support. **The local installation in `D:\claude\trading` keeps running
independently** — this project has its own database and its own jobs.

## Architecture

Single Render web service owning a SQLite database on a persistent disk (`/data`). All data
pulls and prediction jobs run in-process via `src/scheduler.js` as memory-capped child
processes (Render disks can't be shared between services, so separate cron services are not
an option). Schedule: DAMAS 5 min, ENTSO-E 15 min, OPCOM 30 min, weather 60 min, predict
15 min, score hourly, offers daily 03:30 UTC, retrain Sunday 04:00 UTC.

## Deploy

1. Push this folder to a Git repo; create a new **Blueprint** on Render pointing at it
   (`render.yaml` defines the service, disk and env vars).
2. Set the secret env vars in the Render dashboard:
   - `ENTSOE_TOKEN` — ENTSO-E API token
   - `USERS` — `catalin:pw1,georgiana:pw2,iulian:pw3` (pick strong passwords)
   - `SESSION_SECRET` — auto-generated by the blueprint
3. **Migrate history** (predictions/positions are not re-derivable): after the first deploy,
   copy the local database to the disk:
   `render ssh gan-trading` then from local: `scp D:\claude\trading\tool\market.db <ssh-host>:/data/market.db`
   (or use `render disk` snapshots). Restart the service afterwards. Skipping this starts
   the cloud instance with an empty db — market data backfills via `npm run backfill:*`,
   but position/prediction history would be lost.
4. Also copy `model.json` to `/data/model.json` (or let the weekly retrain create it once
   enough data is backfilled).

## Local development

```powershell
npm install
$env:USERS='dev:dev'; node src\server.js     # UI on :8077, jobs off
$env:ENABLE_JOBS='1'                          # enable pulls/predictions locally (own data/ dir)
```

## Users & auth

- Sessions are HMAC-signed cookies (30 days), login at `/login`, sign out via ⎋ in the nav.
- Every position save / unlock / lock records **who** did it (`user` columns in
  `user_bets`, `user_bets_log`, `page_unlocks`).
- `/health` is the only unauthenticated route (Render health checks).

## Mobile

Responsive below 760px; use the **Columns ▾** picker to slim the table per device (the
selection is remembered per browser). Add to home screen for app-like use.

## Costs (approx.)

Standard instance (2 GB) ~$25/mo + 10 GB disk ~$2.5/mo. The offers parse runs with a
1.5 GB heap cap as a child process; if it OOMs on Standard, either disable it
(`JOBS_DISABLED=oferte`) — the model tolerates missing merit-order features — or upsize
the instance.
